# คำอธิบายฟังก์ชันและตัวแปรใน `server.js` แบบละเอียด

เอกสารฉบับนี้อธิบายการทำงานของโค้ดใน `server.js` เรียงลำดับตามฟังก์ชันและการทำงานที่ปรากฏในไฟล์

---

### 1. `requireAuth(req, res, next)` (Middleware ตรวจสอบการเข้าสู่ระบบ)
*   **หน้าที่:** เป็นด่านหน้าคอยตรวจว่าผู้ใช้งานได้ล็อกอินเข้ามาหรือยัง ก่อนจะอนุญาตให้เข้าถึงหน้าเพจหรือ API ต่างๆ
*   **ตัวแปรและที่มา:**
    *   `req.session.user`: ดึงมาจาก Session ที่เก็บไว้ในหน่วยความจำของ Server หลังจากผู้ใช้ Login สำเร็จ
*   **การทำงาน:** ถ้ามี `req.session.user` จะให้ทำงานต่อ (`next()`) ถ้าไม่มีจะตรวจสอบว่าถ้าเป็น API (`/api`) ให้ส่ง Error 401 กลับไป แต่ถ้าเป็นหน้าเว็บปกติจะ Redirect ไปที่หน้า `/login`

---

### 2. `requireAdmin(req, res, next)` (Middleware ตรวจสอบสิทธิ์แอดมิน)
*   **หน้าที่:** ตรวจสอบว่าผู้ใช้ที่ล็อกอินอยู่เป็น "admin" เท่านั้น
*   **ตัวแปรและที่มา:**
    *   `req.session.user.role`: ดึงมาจากข้อมูลผู้ใช้ใน Session
*   **การทำงาน:** ต้องผ่านเงื่อนไขคือต้องมี User และ `role` ต้องเท่ากับ `'admin'` เท่านั้น ถ้าไม่ใช่จะส่ง Status 403 (Forbidden) กลับไป

---

### 3. `requireManager(req, res, next)` (Middleware ตรวจสอบสิทธิ์ผู้จัดการ)
*   **หน้าที่:** อนุญาตเฉพาะผู้ที่มีสิทธิ์เป็น "manager" หรือ "admin" เท่านั้น
*   **ตัวแปรและที่มา:**
    *   `req.session.user.role`: ดึงมาจาก Session
*   **การทำงาน:** คล้าย `requireAdmin` แต่เปิดกว้างกว่าเพื่อให้ผู้จัดการทั่วไปสามารถจัดการสินค้าในคลังได้

---

### 4. `initializeDatabase()` (ฟังก์ชันเตรียมความพร้อมฐานข้อมูล)
*   **หน้าที่:** สร้าง Table ทั้งหมดในไฟล์ `wms.db` กรณีที่รันโปรแกรมครั้งแรกและยังไม่มี Table
*   **ตัวแปรและที่มา:**
    *   `db.run(...)`: คำสั่งภาษา SQL เพื่อสร้างตาราง `Users`, `Products`, `Stock`, `Transactions_Log`, และ `Expired_Alerts`
*   **การทำงาน:** จะรันคำสั่ง `CREATE TABLE IF NOT EXISTS` เพื่อป้องกันการสร้างซ้ำ และในช่วงท้ายจะทำการสร้างบัญชี Admin เริ่มต้น (username: admin, password: 1234) โดยใช้ `bcrypt.hashSync` เพื่อเข้ารหัสรหัสผ่าน

---

### 5. `getBangkokTimestamp()` (ฟังก์ชันดึงเวลาปัจจุบัน)
*   **หน้าที่:** แปลงเวลาของระบบให้เป็นรูปแบบวันที่ของประเทศไทย (GMT+7)
*   **ตัวแปรและที่มา:**
    *   `new Date()`: ดึงเวลาปัจจุบันจากระบบปฎิบัติการ
*   **การทำงาน:** ใช้คำสั่ง `.toLocaleString` กำหนด `timeZone: 'Asia/Bangkok'` และจัดรูปแบบให้เป็น `YYYY-MM-DD HH:MM:SS` เพื่อให้นำไปบันทึกใน Log ได้อ่านง่าย

---

### 6. `app.post('/login', ...)` (Route ตรวจสอบการล็อกอิน)
*   **หน้าที่:** รับข้อมูลชื่อผู้ใช้และรหัสผ่านเพื่อตรวจสอบความถูกต้อง
*   **ตัวแปรและที่มา:**
    *   `username`, `password`: ดึงมาจาก `req.body` (ข้อมูลที่ผู้ใช้พิมพ์มาจากหน้าฟอร์ม)
    *   `user`: ข้อมูลที่ดึงมาจากฐานข้อมูล Table `Users` โดยใช้ชื่อ `username` เป็นตัวค้นหา
*   **การทำงาน:** นำ `password` จากหน้าฟอร์มมาเปรียบเทียบกับรหัสที่เข้ารหัสไว้ใน DB (`user.password`) ด้วยคำสั่ง `bcrypt.compareSync` ถ้าตรงกันจะเก็บข้อมูลใส่ `req.session.user` และส่งผู้ใช้ไปหน้า Dashboard

---

### 7. `app.get('/api/inventory', ...)` (Route ดึงข้อมูลสต็อกทั้งหมด)
*   **หน้าที่:** ประมวลผลและส่งข้อมูลสินค้าพร้อมจำนวนล็อต (Batch) ไปแสดงที่หน้าเว็บ
*   **ตัวแปรและที่มา:**
    *   `query`: คำสั่ง SQL สำหรับดึงข้อมูล `Products` มา Join กับ `Stock`
    *   `rows`: ผลลัพธ์จากการ Query ที่เป็นรายการล็อตย่อยๆ ทั้งหมด
    *   `grouped`: ตัวแปร Object ที่สร้างขึ้นเพื่อรวบรวมล็อตย่อยๆ ให้เข้าไปอยู่ภายใต้สินค้าตัวเดียวกัน (Product Grouping)
    *   `todayStr`: วันที่ปัจจุบันในรูปแบบ String เพื่อนำไปเทียบว่าล็อตไหนหมดอายุ
*   **การทำงาน:** เมื่อได้ข้อมูลมา จะวนลูปแยกประเภท ล็อตที่หมดอายุแล้วจะบวกเข้า `expired_quantity` ล็อตที่ยังไม่หมดอายุจะบวกเข้า `total_quantity` เพื่อส่งไปให้ Frontend แสดงผล

---

### 8. `autoLogExpiredProducts(callback)` (ฟังก์ชันบันทึกสินค้าหมดอายุอัตโนมัติ)
*   **หน้าที่:** ตรวจหาล็อตสินค้าที่หมดอายุแล้ว แต่ยังไม่ได้บันทึกประวัติการหมดอายุ
*   **ตัวแปรและที่มา:**
    *   `qFindExpired`: SQL ดึงข้อมูลล็อตที่หมดอายุ (expiry_date < วันนี้) และยังมีสินค้าเหลืออยู่
    *   `qCheckLog`: SQL สำหรับตรวจสอบว่าล็อตพะวกนี้เคยถูกบันทึกไปหรือยัง (ป้องกันการบันทึกซ้ำ)
    *   `loggedStockIds`: ตัวเก็บ ID ล็อตที่สรุปแล้วว่าบันทึกแล้ว
*   **การทำงาน:** จะดึงข้อมูลมาเปรียบเทียบกัน ล็อตไหนที่เพิ่งหมดอายุและยังไม่มี Log จะทำการสร้าง Log ใหม่ด้วย `action_type = 'EXPIRED'` และกำหนดผู้ทำรายการเป็น `'System/Auto'`

---

### 9. `app.post('/api/stock/add', ...)` (Route เพิ่มสต็อกสินค้า)
*   **หน้าที่:** บันทึกการรับเข้าสินค้าใหม่และคำนวณวันหมดอายุให้อัตโนมัติ
*   **ตัวแปรและที่มา:**
    *   `product_id`, `receive_date`, `quantity`: รับมาจาก `req.body`
    *   `shelf_life_days`: ดึงมาจาก Table `Products` เพื่อดูว่าสินค้าชนิดนี้มีอายุกี่วัน
    *   `expiry_date`: เกิดจากการนำ `receive_date` มาบวกเพิ่มด้วย `shelf_life_days`
*   **การทำงาน:** จะเริ่มทำ `TRANSACTION` ในฐานข้อมูล เพื่อบันทึกข้อมูลลงใน Table `Stock` และบันทึกประวัติลงใน `Transactions_Log` พร้อมๆ กัน

---

### 10. `app.post('/api/stock/withdraw', ...)` (Route เบิกสินค้า)
*   **หน้าที่:** ตัดสต็อกสินค้าโดยใช้หลักการ **FIFO (First-In, First-Out)** คือตัดล็อตที่เก่าที่สุดก่อน
*   **ตัวแปรและที่มา:**
    *   `product_id`, `quantity`: รับมาจาก `req.body`
    *   `batches`: รายการล็อตทั้งหมดของสินค้านั้นๆ ที่ดึงมาจาก DB เรียงลำดับตามวันที่รับเข้า
    *   `qtyToWithdraw`: ตัวเลขจำนวนที่ผู้ใช้ต้องการเบิก
*   **การทำงาน:** วนลูปตรวจสอบล็อตที่เก่าที่สุด ถ้าจำนวนในล็อตนั้นมีพอจะตัดจนหมด ก็จะ `DELETE` ล็อตนั้นทิ้ง ถ้าเหลือก็จะ `UPDATE` ยอดที่เหลือ จนกว่าจำนวนที่เบิกจะครบตามที่ต้องการ

---

### 11. `app.post('/api/stock/delete-expired', ...)` (Route ลบสินค้าที่หมดอายุ)
*   **หน้าที่:** ลบข้อมูลล็อตสินค้าที่หมดอายุออกจากฐานข้อมูลตามที่ผู้ใช้เลือกมา
*   **ตัวแปรและที่มา:**
    *   `expired_batches`: รับมาจาก `req.body` เป็น Array ของรายการล็อตที่ต้องการลบ
*   **การทำงาน:** วนลรูปสั่ง `DELETE` ล็อตสินค้าใน Table `Stock` และบันทึกประวัติใน `Transactions_Log` ว่ามีการลบทิ้งเนื่องจากหมดอายุโดยระบุชื่อสินค้ากำกับไว้ใน Log ด้วย

---

### 12. `app.get('/api/admin/dashboard-stats', ...)` (Route ข้อมูลสถิติแดชบอร์ด)
*   **หน้าที่:** คืนค่าสถิติเชิงลึกสำหรับหน้า Admin (สถิติ 5 ด้าน)
*   **ตัวแปรและที่มา:**
    *   `qProducts`, `qLowStock`, `qValue`, `qUsers`, `qCategoryDist`: ชุดคำสั่ง SQL หลายตัวที่รวมรวมค่าสถิติต่างๆ
*   **การทำงาน:** รันคำสั่ง SQL ทั้งหมดแบบซ้อนกัน (Nest Call) เพื่อสรุปยอดรวมสินค้า, ยอดมูลค่าคลังสินค้า (เอาราคาคูณจำนวน), และสัดส่วนหมวดหมู่สินค้ายอดนิยม เพื่อส่งข้อมูลไปวาดกราฟ Donut Chart บนหน้าเว็บ

---

### 13. `app.post('/api/admin/reseed-stock', ...)` (Route สุ่มสต็อกสินค้า)
*   **หน้าที่:** ลบสต็อกเก่าและสร้างสต็อกใหม่แบบสุ่ม (เอาไว้ใช้ทดสอบระบบหรือ Demo)
*   **ตัวแปรและที่มา:**
    *   `Math.random()`: ใช้สุ่มจำนวนสินค้า (10-150 หน่วย) และสุ่มวันรับเข้าลดถอยหลัง (0-4 วัน)
*   **การทำงาน:** เคลียร์ Table `Stock` และ `Transactions_Log` จนเกลี้ยง แล้วดึงสินค้าทุกตัวในระบบมาทำการ "เติมสต็อกแบบสุ่ม" เข้าไปใหม่ทั้งหมด

---

### 14. `app.delete('/api/admin/products/:id', ...)` (Route ลบสินค้าออกจากระบบ)
*   **หน้าที่:** ลบรายการสินค้าอออกจากระบบ โดยมีเงื่อนไขความปลอดภัย
*   **ตัวแปรและที่มา:**
    *   `id`: รับมาจาก Parameter ของ URL (`:id`)
    *   `stockRow.total_qty`: คำนวณหายอดคงเหลือของสินค้านี้ก่อนลบ
*   **การทำงาน:** ระบบจะไม่อนุญาตให้ลบถ้ายังมีสินค้าเหลืออยู่ในสต็อก (แม้แต่ 1 ชิ้น) ต้องเบิกออกให้หมดก่อน เมื่อลบสำเร็จ จะทำการอัปเดตประวัติเก่าๆ ให้ไม่หายไปแต่สถานะของ `product_id` จะกลายเป็น `NULL` พร้อมกำกับชื่อสินค้าไว้ในข้อมูลแทน
