# 📦 WMS (Warehouse Management System)

ระบบจัดการคลังสินค้าออนไลน์ขนาดย่อม (Warehouse Management System) ที่ออกแบบมาให้ใช้งานง่าย รวดเร็ว และมีหน้าตาทันสมัย เหมาะสำหรับการจัดการคลังสินค้าทั่วไป การติดตามการเพิ่ม/เบิกสินค้า และการจัดการข้อมูลวันหมดอายุ

---

## ⚡ คุณสมบัติเด่น (Features)
- 📊 **Interactive Dashboard**: สรุปยอดรวมสินค้าในคลัง แจ้งเตือนสินค้าใกล้หมดอายุ และกราฟสถิติรายเดือน
- 📦 **Inventory Management**: จัดการข้อมูลสินค้า ดูยอดคงเหลือ และระยะเวลาจัดเก็บ (Shelf Life)
- 🔄 **Stock Operations**: จำลองการนำเข้า (Add) ดึงออก (Withdraw) พร้อมระบบตัดสต็อกแบบ FIFO และสุ่มลอจิกอัตโนมัติ
- 📜 **Historical Logs**: บันทึกประวัติการทำรายการทุกขั้นตอนตลอดระยะเวลาที่ผ่านมา
- 👥 **User Roles**: แยกสิทธิ์เข้าถึงระหว่าง `manager` (จัดการรหัสผ่าน/ผู้ใช้) และ `staff` (จัดการสินค้าทั่วไป)
- 🚀 **Auto-Start**: แค่สั่งรัน ระบบจะเปิดหน้าเว็บให้คุณโดยอัตโนมัติทันที!

---

## 🛠️ โครงสร้างทางเทคโนโลยี (Tech Stack)

### **Backend**
- **Node.js** Environment สำหรับการทำงานหลังบ้าน
- **Express.js** Framework สำหรับสร้าง API และ Routing
- **SQLite3** ฐานข้อมูลแบบไฟล์เดียว เคลื่อนย้ายง่าย ไม่ต้องติดตั้งโปรแกรม DB แยก

### **Frontend & Template**
- **EJS (Embedded JavaScript)** ใช้เรนเดอร์ HTML Template แบบไดนามิก
- **Tailwind CSS (via CDN)** เฟรมเวิร์ก CSS สำหรับออกแบบให้ดูทันสมัย สวยงาม ไร้กรอบ
- **Chart.js** ไลบรารีสำหรับวาดกราฟสถิติที่สวยงามบน Dashboard
- **SweetAlert2** ป็อปอัปแจ้งเตือนสวยๆ

## 🚀 วิธีเปิดใช้งาน (Quick Start)

เพื่อให้สะดวกและเปิดโปรแกรมได้ทันทีโดยไม่ต้องจำคำสั่งติดตั้ง แค่เปิด terminal และรันคำสั่งเดียวจบ:

**1. เริ่มระบบ (Start Server & Auto-install)**
```bash
npm start
```

> 💡 **เมื่อคุณรันคำสั่งนี้**: ระบบจะทำการ **ดาวน์โหลด Dependencies ทุกอย่างให้อัตโนมัติ (npm install)** จากนั้นจะสตาร์ทเซิร์ฟเวอร์ และ **เด้งเปิดเบราว์เซอร์เข้าหน้าแอปพลิเคชัน** ให้คุณทันที! โดยไม่ต้องเข้าพิมพ์ที่อยู่ URL เอง

## 🔑 บัญชีเข้าสู่ระบบพื้นฐาน (Default Accounts)

ระบบจะสุ่มหรือสร้างผู้จัดการ (Manager) ให้เป็นค่าเริ่มต้น

- **Password:** `1234`

(หลังจากเข้ามาในระบบ คุณสามารถไปที่เมนู **"จัดการผู้ใช้งาน"** เพื่อสร้างบัญชีให้ทีมงาน (Staff) เพิ่มเติมได้)

## 📂 โครงสร้างโปรเจกต์
- `server.js` - โค้ดหลังบ้าน (Backend Server) กำหนด Routing, Database, Business Logic 
- `wms.db` - ฐานข้อมูล SQLite เก็บข้อมูลและรายการจำลองทั้งหมด 
- `products.csv` - ข้อมูลสินค้าเบื้องต้น ที่ถูกแปลงเพื่อให้มีสมมติฐานการหมดอายุที่สมจริง
- `views/` - ที่เก็บหน้าต่างโปรแกรมแบบ EJS ทั้งหมด เช่น Dashboard, Inventory ฯลฯ
- `public/` - ที่เก็บไฟล์ Static Assets เพิ่มเติม (เช่น รูปภาพ, CSS แยก)
