# ระบบจัดการคลังสินค้า (Warehouse Management System - WMS)

โปรเจกต์นี้เป็นวิทยานิพนธ์/โครงการในรายวิชาการพัฒนาเว็บแอปพลิเคชัน พัฒนาขึ้นเพื่อจำลองระบบจัดการคลังสินค้าที่ใช้งานได้จริงในรูปแบบ Local Web Application มุ่งเน้นไปที่ความง่ายในการใช้งาน ความปลอดภัยของข้อมูล และความถูกต้องของการจัดการสต็อกแบบ FIFO

## 📌 คุณสมบัติหลัก (Key Features)

- **Real-time Inventory Dashboard:** ติดตามสถานะสินค้าคลังภาพรวม มูลค่าสต็อก และสินค้าใกล้หมดในหน้าเดียว
- **User Roles & RBAC:** ระบบจัดการสิทธิ์ผู้ใช้งาน แบ่งเป็น Admin, Manager และ Staff พร้อมระบบรักษาความปลอดภัยในการเข้าถึง API
- **Stock Management (FIFO):** ระบบตัดสต็อกอัตโนมัติโดยยึดตามล็อตที่เข้าก่อน-ออกก่อน เพื่อป้องการสินค้าค้างสต็อกหรือเสื่อมสภาพ
- **Transaction History:** บันทึกประวัติการทำรายการทุกขั้นตอนอย่างละเอียด (Add, Withdraw, Expired)
- **Expired Alerts:** ระบบแจ้งเตือนสินค้าหมดอายุล่วงหน้า และบันทึกประวัติสินค้าหมดอายุลงในระบบ Log อัตโนมัติ

## 🛠️ เทคโนโลยีที่ใช้ (Technology Stack)

- **Frontend:** HTML5, CSS (Tailwind), JavaScript, และ **EJS Templates**
- **Backend:** **Node.js** ร่วมกับ **Express.js**
- **Database:** **SQLite** (จัดเก็บข้อมูลเป็นไฟล์ `wms.db` ภายในโปรเจกต์)
- **Security:** Session-based Authentication และ Password Hashing ด้วย `bcryptjs`

## 📊 โครงสร้างฐานข้อมูล (Database Schema)

ระบบใช้งานฐานข้อมูล SQLite โดยมี 5 ตารางหลักดังนี้:

1. **USERS:** ข้อมูลผู้ใช้และสิทธิ์การเข้าใช้งาน (Role-based)
2. **PRODUCTS:** ข้อมูลพื้นฐานสินค้า ราคา และอายุการเก็บรักษา (Shelf Life)
3. **STOCK:** ข้อมูลจำนวนสินค้าแยกตามล็อต วันที่รับ และวันหมดอายุ
4. **TRANSACTIONS_LOG:** ประวัติการเคลื่อนไหวของสต็อกและการแก้ไขข้อมูล
5. **EXPIRED_ALERTS:** ข้อมูลสินค้าที่หมดอายุแล้วเพื่อใช้ตรวจสอบย้อนหลัง

## 🚀 ขั้นตอนการติดตั้งและใช้งาน (Installation)

สำหรับการตรวจโปรเจกต์ (สำหรับอาจารย์) สามารถติดตั้งและรันได้ง่ายผ่าน **Batch Script** ดังนี้:

1. **ติดตั้ง Node.js:** กรุณาตรวจสอบว่าเครื่องคอมพิวเตอร์มีการติดตั้ง Node.js เรียบร้อยแล้ว
2. **เตรียมความพร้อม:** แตกไฟล์โปรเจกต์ลงในโฟลเดอร์ที่ต้องการ
3. **รันระบบด้วย Script:** ดับเบิลคลิกที่ไฟล์:
   ```bash
   start_project.bat
   ```
   *Script นี้จะทำการติดตั้ง dependencies ต่างๆ และเริ่มต้น Server ให้โดยอัตโนมัติ*
4. **เข้าใช้งาน:** ระบบจะเปิด Browser ไปที่ `http://localhost:3000` โดยอัตโนมัติ (หากไม่เปิด สามารถพิมพ์ URL เองได้)

## 🔑 ข้อมูลการเข้าสู่ระบบเริ่มต้น (Default Admin Credentials)

อาจารย์สามารถใช้ข้อมูลด้านล่างนี้เพื่อล็อกอินและทดสอบฟังก์ชันทั้งหมดของระบบ:

- **Username:** `admin`
- **Password:** `1234`

## 🛠️ การแก้ไขปัญหาเบื้องต้น (Troubleshooting)

- **Script ไม่ทำงาน:** ตรวจสอบให้แน่ใจว่าได้ติดตั้ง Node.js และเพิ่มลงใน System Path แล้ว
- **Port 3000 ถูกใช้งานอยู่:** หากรันหน้าเว็บไม่ได้ ให้ตรวจสอบว่าไม่มีโปรแกรมอื่นใช้งาน Port 3000 อยู่ หรือปิดและรัน `start_project.bat` ใหม่อีกครั้ง
- **ข้อมูลไม่แสดง:** หากพบปัญหาระบบฐานข้อมูล ให้ตรวจสอบว่าไฟล์ `wms.db` อยู่ในโฟลเดอร์หลักของโปรเจกต์
